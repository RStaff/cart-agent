7. `overflow-list` receives all menu children and manages overflow with `minimum-items: 2`.
8. `assets/header-menu.js` powers hover/focus activation, overflow submenu behavior, submenu-height calculation, and image preloading.

## Mobile Drawer Flow

Primary files:

- `blocks/_header-menu.liquid`
- `snippets/header-drawer.liquid`
- `snippets/menu-font-styles.liquid`
- `snippets/submenu-font-styles.liquid`
- `assets/header-drawer.js`
- `assets/icon-menu.svg`

Render sequence:

1. `header.liquid` captures `drawer_menu` through `_header-menu` with `variant: mobile`.
2. `header-row.liquid` places `drawer_menu` as the first top-row item.
3. `_header-menu.liquid` renders `header-drawer` with `data_header_drawer_type: mobile-drawer`.
4. `header-drawer.liquid` loads `assets/header-drawer.js`.
5. The drawer uses a root `details` element with `id="Details-menu-drawer-container"`.
6. The trigger `summary` toggles the drawer and inlines `icon-menu.svg` and `icon-close.svg`.
7. The drawer renders a `nav` with `aria-label` from the header navigation translation.
8. Link rendering branches for menus with fewer than three levels and menus with three levels.
9. Accordion behavior is enabled by block settings, but the current source configuration sets `drawer_accordion` to `false`.
10. Drawer utility links can render localization controls when country or language selectors are available.

`assets/header-drawer.js` source-proves:

- Escape closes the drawer.
- Open and close update `aria-expanded`.
- Open adds `menu-open`.
- Submenu navigation uses `header-drawer/open/submenu` and `header-drawer/back`.
- Focus is trapped after drawer animation.
- Open nested details are reset when the drawer closes.

## Global-Shell Authority Boundaries

The header is a global shell surface. Source-level changes would affect multiple page types because the header group is mounted in the theme group, not in a single product, collection, or cart template.

Exercise 007 is analysis-only. No change is authorized to:

- Header group JSON
- Header section Liquid
- Header menu block
- Header drawer
- Header actions
- Search modal
- Predictive-search behavior
- Announcement bar
- Menu data
- Account, localization, or cart action behavior
- Any JavaScript asset

## Reusable ShopiFixer Patterns

- Treat Shopify header groups as global-shell authority, not page-local authority.
- Separate JSON composition from Liquid rendering and JavaScript behavior before recommending any change.
- Confirm desktop and mobile navigation ownership separately; they share menu data but use different render and interaction paths.
- Treat static theme blocks as part of the authoritative render chain.
- Distinguish archived rendered evidence from current live runtime behavior.
- Do not treat a menu handle as proof of current live menu items without current runtime or admin-backed evidence.
- Header action inventory should identify account/cart/localization surfaces without re-certifying cart, account, or checkout behavior.
- For global-shell surfaces, rollback planning must account for JSON group settings, Liquid snippets/blocks, and JavaScript assets.

## Risks

- Header group JSON is auto-generated and can drift through Shopify admin/theme editor changes.
- Header behavior spans JSON, Liquid, CSS, JavaScript, and Shopify object data.
- Desktop and mobile navigation use different interaction paths, so a source change in one area may not validate the other.
- Search and predictive-search rely on section rendering, routes, DOM morphing, and recently viewed state.
- Cart and account actions are embedded in the header but have authority boundaries outside Exercise 007.
- Sticky header behavior affects global page layout and can change perceived navigation stability across all page types.
- Mega-menu featured content can depend on collection/product objects not proven in this inventory.
