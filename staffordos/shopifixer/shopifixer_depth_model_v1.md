# SHOPIFIXER DEPTH MODEL V1

## Purpose

Define how deep ShopiFixer goes when connected to screenshots, Shopify theme source, and codebase structure.

This prevents under-selling the value and prevents over-scoping the work.

## Core Principle

ShopiFixer is not just a visual audit.

ShopiFixer connects:

visual evidence
+
theme/source structure
+
commerce path
+
implementation target
+
proof package
+
learning system

## Standard $950 Fix Sprint

The standard ShopiFixer sprint includes deep enough inspection to execute ONE scoped fix.

Included:

- desktop screenshot review
- mobile screenshot review
- Shopify theme fingerprint
- homepage template inspection
- section/order detection
- affected section identification
- product/CTA/trust validation for selected issue
- implementation plan for selected issue
- one scoped implementation
- QA
- before/after screenshots
- proof package

Not included:

- full site rebuild
- full theme refactor
- multi-page redesign
- app conflict remediation
- checkout customization
- SEO campaign
- ad campaign
- long-term CRO retainer
- complete codebase cleanup

## Extended Discovery

Use when the issue appears broader than one sprint or the correct sprint is not yet obvious.

Extended Discovery may include:

- multi-page funnel review
- product page review
- collection page review
- cart review
- checkout path review where accessible
- theme architecture review
- app/script risk review
- performance red flags
- accessibility red flags
- tracking/analytics readiness
- prioritized fix map

Extended Discovery does not automatically include implementation of every finding.

## Implementation Depth Levels

Level 1:
Theme editor setting change.

Level 2:
Section reorder, product binding, copy, CTA, trust placement.

Level 3:
Small custom Liquid section.

Level 4:
Theme code modification across section/snippet/CSS.

Level 5:
Out of standard sprint scope. Requires custom quote.

## $950 Boundary

A $950 sprint should usually be Level 2 or Level 3.

Level 1 may be acceptable only when the diagnosis, QA, and proof package clearly justify the value.

Level 4 may be accepted only when tightly scoped to one visible issue.

Level 5 is not included.

## Why This Is Worth Paying For

The merchant is not paying for superficial copy edits.

The merchant is paying for:

- diagnosis
- theme/source mapping
- scoped implementation
- QA
- proof

ShopiFixer should be able to answer:

1. What is wrong?
2. Why does it matter?
3. Where does it live in the Shopify theme?
4. What is the smallest safe fix?
5. How will success be proven?

## ShopiFixer Learning System

Every completed sprint should generate reusable intelligence.

Capture:

- merchant/store
- theme
- theme version or theme ID
- detected pattern
- implementation level
- affected template
- affected section
- files touched
- time spent
- QA results
- before evidence
- after evidence
- proof package quality
- merchant reaction
- revenue generated

## Pattern Library

Every completed sprint should either match an existing pattern or create a new pattern.

Initial examples:

Pattern 001:
Homepage Product Value Path

Pattern 002:
Trust Gap Near CTA

Pattern 003:
Weak Featured Collection Merchandising

Pattern 004:
Cart Confidence Gap

Pattern 005:
Mobile Product Discovery Failure

Each pattern should eventually include:

- use cases
- screenshot signals
- theme signals
- implementation paths
- QA checklist
- proof requirements
- pricing boundary

## Theme Library

Track common Shopify themes and their recurring structures.

Examples:

- Horizon
- Dawn
- Refresh

For each theme, capture:

## AI + Human Model

AI/Codex can inspect, map, draft, and accelerate.

Ross approves:

- selected issue
- claim quality
- scope
- merchant-facing recommendation
- implementation boundary
- final proof package

## Anti-Commodity Rule

Merchants are not paying for AI.

Merchants are paying for:

- diagnosis
- implementation
- QA
- proof
- confidence

The intelligence system exists to improve delivery quality, not replace judgment.

## Anti-Drift Rule

Do not sell deep engineering under the $950 fixed sprint unless the scope is still one visible, provable issue.

Do not call superficial changes conversion work.

Every ShopiFixer sprint must connect visible friction to a real implementation target and proof.

## Success Standard

A completed sprint should make future audits:

- faster
- more accurate
- easier to scope
- easier to implement
- easier to prove
