# StaffordOS Loop v1

## Purpose

StaffordOS Loop v1 connects the existing governance pieces into one real control path.

It does not replace existing agents, gates, routers, or product code.

## Loop

request → classify → owner check → gate decision → packet preparation → execution through existing agents → verification → truth update

## Required Owners

- agent registry
- run agent entrypoint
- execution driver
- progress contract
- contract validator
- revenue/goal gate
- safe write helper
- system truth file
- source/runtime policy

## Rule

No product or production change proceeds unless StaffordOS Loop v1 can classify it, locate its owners, require verification, and require truth update.

## Current Status

This is the first real loop controller. It is not autonomous self-healing yet. It is the control spine that makes self-healing possible.
